CSS 选择器对比

理解 .marked p 和 p.marked 的区别，是掌握 CSS 选择器的关键一步。

两个选择器，截然不同的含义

.marked p

.marked p {
  color: blue;
}

读作："class 为 marked 的元素内部的所有 p 元素"
关系：祖先 → 后代
空格表示后代选择器

p.marked

p.marked {
  color: green;
}

读作："所有同时是 p 元素且 class 为 marked 的"
关系：同一个元素
连写表示交集选择器（既是 p 又是 .marked）

一句话区分

.marked p — 从父元素找子元素：先找到 marked 类，再找里面的 p。两个元素，一内一外。

p.marked — 从元素本身找类：必须是 p 标签，且这个 p 标签上必须有 marked 类。同一个元素满足两个条件。

HTML 结构

以下面这段 HTML 为例，看看两个选择器分别选中了什么

<div class="container">
  <div class="marked"> ← .marked 父容器
    <p>这是一个 p 标签</p> ← .marked p 选中 (1)
    <p class="marked">这是一个 p.marked</p> ← 两个都选中 (1+2)
    <span>这是一个 span</span> ← 两个都不选中
  </div>
  <p class="marked">这里的 p.marked</p> ← p.marked 选中 (2)
</div>
■ (1) .marked p 选中   ■ (2) p.marked 选中   ■ 都选中

.marked p 后代选择器

.marked p { color: #6366f1; }

CSS 理解：先找到 class="marked" 的元素，再找它内部（所有层级）的 <p> 标签。
p 不一定要是 marked 的直接子元素，只要是后代就行。marked 元素本身的样式不会被改变。

<div class="container">
<div class="marked">

我是 marked 内部的 p → 会被 .marked p 选中

我也是 marked 内部的 p，同时我自己也有 marked 类

我是 span，选不中
<div>（再嵌套一层）

我是更深层的 p，同样会被 .marked p 选中！

我在 marked 外面，是 container 的直接子元素 → 选不中

我在 marked 外面，但我有 marked 类 → p.marked 能选中我

p.marked 交集选择器

p.marked { background: #d1fae5; }

CSS 理解：选择同时满足两个条件的元素：必须是 <p> 标签，且必须有 class="marked"。
中间没有空格，表示"且"的关系 —— 同一个元素既要满足标签名条件，又要满足类名条件。
与 .marked p 不同：p.marked 不关心这个 p 在谁里面，只关心"p 自己"有没有 marked 类。

<div class="container">
<div class="marked">

普通 p（无 marked 类）→ 选不中

p 且 class="marked" → p.marked 选中

span 且 class="marked" → 选不中（不是 p）

我在最外层，但我是 p 且有 marked 类 → p.marked 也能选中我

span 且有 marked → 选不中

终极对比表

.marked p

写法.marked p
分隔符空格（后代选择器）
涉及元素两个元素（祖先 + 后代）
选中对象marked 内部的 所有 p
读法marked 里面的 p

p.marked

写法p.marked
分隔符无（交集/类选择器）
涉及元素一个元素（自身）
选中对象同时是 p 且有 marked 类的
读法带 marked 类的 p

记忆口诀

有空格 → 找后代（从父到子，两个标签）

无空格 → 找交集（同一个标签，多个条件）

💡 记住这个核心区别，以后遇到 .a .b、div.a、div.a.b 等组合都能理解。
