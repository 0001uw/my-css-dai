CSS 盒子模型

CSS 盒子模型（Box Model）是网页布局的核心概念。每个 HTML 元素都可以看作一个矩形盒子，由内到外依次为：内容 → 内边距 → 边框 → 外边距。轮廓则悬浮在边框之外。

关系总结

Content → 内容区域，显示文本、图片等实际内容
Padding → 内边距，内容与边框之间的间距，撑大盒子但保持内容干净
Border → 边框，包裹内容和内边距，盒子的"墙壁"
Margin → 外边距，边框之外的间距，控制盒子与周围元素的距离
Outline → 轮廓，绘制在边框之外但不占用空间，不影响布局
从内到外的层次：Content 在最里面 → Padding 包裹 Content，创造呼吸空间 → Border 是盒子的边界墙 → Margin 在 Border 之外，决定盒子与其他元素的距离。

关键理解：Padding 和 Border 会撑大盒子的占地面积，Margin 则是在盒子之外推开其他元素。而 Outline 比较特殊——它画在 Border 外面，但不占空间，不会影响布局。

公式：盒子总宽度 = width + padding-left + padding-right + border-left + border-right
盒子总高度 = height + padding-top + padding-bottom + border-top + border-bottom
（Margin 不计入盒子尺寸，只影响与其他元素的间距）

CSS 三角形原理 提示框中小箭头的构造方法

鼠标悬停在首页的「盒子模型」按钮上时，出现的提示框下方有一个小三角箭头。 这个箭头不是图片，而是用 CSS 的 border（边框）构造出来的。

构造过程

/* 1. 一个宽高为 0 的元素，四周都设边框 */ .arrow { width: 0; height: 0; border: 12px solid black; } /* 2. 三边变透明 → 只剩三角形 */ .arrow-down { border-color: transparent; border-top-color: black; }
四个边框都可见
红 右蓝 下绿 左黄
三边透明 → 三角形
只保留上边框颜色
原理：元素的四个边框在交汇处是斜角连接的。当元素宽高为 0 时，四边的边框在中心点相遇，每一条边都呈现为三角形。

为什么隐藏三边，最后剩下的是三角形？当左、右、下三边的边框都设为透明时，它们就消失了。只给上边设置颜色，这条边的着色区域刚好是一个三角形——剩下三条边的透明区域在视觉上消失，只留下这个有颜色的三角形。

改变方向： border-top-color → 向下 border-bottom-color → 向上 border-left-color → 向右 border-right-color → 向左
